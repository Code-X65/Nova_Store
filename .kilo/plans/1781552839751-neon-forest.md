# Nova_Store Gap Analysis Plan

## 1. What is Well Done

### Backend Architecture
- **Clean layered architecture**: Models → Services → Controllers with clear separation of concerns
- **Comprehensive feature set**: Auth (JWT + OAuth + session), RBAC, products/variants/attributes, cart, wishlist, orders, returns, checkout, coupons, payments (Stripe + Paystack), inventory reservations, shipping, notifications (email/SMS/in-app), audit logging, analytics
- **Security**: Helmet, CORS, rate limiting (per-route), XSS sanitization, idempotency keys, bcrypt password hashing, refresh token rotation, admin lockout, session management with PostgreSQL store
- **Database**: 43 SQL migration files covering full schema, seed data, RPC functions (RPC SQL files exist), RLS policies
- **Infrastructure**: Redis for rate limiting + notification queue, Winston logging, node-cron for cleanup jobs, Swagger docs, health checks
- **Developer experience**: .env.example present, Jest test setup with supertest, nodemon for dev

### Code Quality Patterns
- Consistent use of async/await
- Input validation with Joi/Zod
- Centralized error handling middleware
- Request ID tracking
- Soft delete patterns in some models
- Comprehensive RBAC tables (roles, permissions, user_roles, role_permissions)

## 2. What is Incomplete and Needs Adjustment

### Critical Bugs
| Issue | Location | Fix Required |
|-------|----------|--------------|
| SessionModel import error | `src/jobs/cleanup.job.js:1` | Change `require('../models/session.model')` to `require('../models/session.model').default` or adjust export |
| Security config mismatch | `src/config/security.js` vs `src/services/auth.service.js` | Align refresh token expiry: config says 30d for admin, code implements 8h |
| Admin login checks wrong table | `src/services/auth.service.js` | Admin login uses `userModel.findByEmail()` but `admins` table is separate from `users` |
| Missing DATABASE_URL in .env.example | `.env.example` | Add `DATABASE_URL` for connect-pg-simple session store |

### Architecture Issues
- **RBAC not enforced**: Admin routes use `requireAdmin` middleware which grants full access instead of checking specific permissions via `hasPermission`. Some routes have commented-out permission checks.
- **Dual auth model confusion**: Both `users` and `admins` tables exist. `auth.service.js` checks `users` for admin login, but `admin.model.js` queries `admins`. Needs consolidation or clear documentation.
- **Hardcoded values**: 
  - `admin@novastore.com` hardcoded in auth.service
  - Default shipping costs in shipping.service
  - 7.5% tax rate in checkout.service
- **Missing DATABASE_URL**: Session store requires `DATABASE_URL` but .env.example omits it

### Missing RPC Functions
Code references these Supabase RPCs but many are missing from SQL:
- `create_order_with_items` - referenced in order.service
- `reserve_stock_increment` / `release_stock_reservation` / `commit_reserved_stock` / `release_session_reservations` - inventory.service
- `increment_coupon_usage` - coupon.service
- `is_coupon_valid_for_user` - coupon.service
- `get_products_by_attributes` - product.service (SQL exists: 033_attribute_filter_rpc.sql)
- `get_sales_summary`, `get_best_sellers`, `get_user_growth` - analytics (SQL exists: 017_analytics_views.sql)

### Test Coverage Gaps
Existing tests: auth, category, product, coupon, SMS, idempotency, maintenance, middleware
Missing: order, cart, checkout, payment, inventory, shipping, notification, review, admin controllers

### Documentation Gaps
- No README with setup instructions
- No architecture diagram or ADR
- Swagger exists but may be incomplete
- No deployment guide (Docker, CI/CD)

## 3. What Doesn't Exist but is Essential

### Frontend (Highest Priority)
Current state: Empty Vite + React template with only logo/count button
Essential missing:
- Authentication pages (login, register, forgot password, OAuth callbacks)
- Product catalog (listing, search, filter by attributes, category navigation)
- Product detail page (variants, images, reviews)
- Shopping cart page
- Wishlist page
- Checkout flow (shipping, payment, order confirmation)
- User profile (addresses, order history, returns)
- Admin dashboard (product management, order management, analytics, user management, coupon management, settings)
- Responsive design system / component library

### Missing API/Feature Endpoints
- **Image upload/CDN**: Products reference images but no upload endpoint or CDN configuration
- **Email tracking**: No open/click tracking for notifications
- **WebSocket/real-time**: No real-time order status updates or notifications
- **Search**: No full-text search endpoint (Algolia/Meilisearch integration)
- **File attachments**: No support for product manuals, return evidence photos
- **Multi-currency**: Hardcoded single currency
- **Tax calculation**: Hardcoded 7.5% instead of configurable tax rules by region
- **Invoice generation**: No PDF invoice for orders
- **Email templates management**: No admin UI for editing notification templates

### DevOps / Production Essentials
- **Dockerfile + docker-compose**: No containerization
- **CI/CD pipeline**: No GitHub Actions / deployment workflow
- **Database seeding script**: No npm script for running migrations + seeds
- **Log aggregation**: Winston logs to console only, no ELK/DataDog integration
- **Health check depth**: Basic status only, no DB/Redis dependency checks
- **Backup strategy**: No documented backup/restore procedure for Supabase

### Security Essentials
- **CSP nonce**: Current CSP is static; should use nonces for inline scripts
- **CSRF protection**: express-session used but no CSRF tokens
- **Secret rotation**: JWT secrets, session secrets have no rotation mechanism
- **Penetration testing**: No evidence of security audit
- **GDPR/compliance**: No data export/deletion endpoints beyond basic account deletion
- **apiKey for Stripe/Paystack**: Keys in .env but no environment-based key rotation

### Missing Tests
- Zero integration tests for cart, checkout, orders, payments, inventory
- No E2E tests
- No load/performance tests
- No mutation/chaos testing for Supabase failover
