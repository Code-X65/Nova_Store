# Jest API Testing Plan

## Current State Analysis

**Existing Setup:**
- Jest v30.3.0 + Supertest v7.2.2 installed
- Existing test patterns in `tests/unit/` and `tests/integration/`
- Integration tests use Supertest with Express app directly
- Rate limiting middleware applied to routes (could slow tests)
- Redis client connection management needed in afterAll hooks

**Key Challenges:**
- Supabase database dependency (requires mocking for fast tests)
- Email verification blocking login flows
- Rate limiting interfering with rapid test execution
- OAuth callbacks requiring external service mocking

## Testing Strategy

### 1. Test Categories

| Type | Purpose | Speed | Database |
|------|---------|-------|----------|
| Unit | Service/model logic | <50ms | Mocked |
| Integration | Route + middleware + service | <500ms | Mocked or test DB |
| E2E (optional) | Full stack with real DB | <2s | Test DB |

### 2. Test Environment Setup

**File: `tests/setup.js`** (enhance existing `setup.js`)
```javascript

jest.mock('../src/middlewares/rate-limit.middleware', () => ({
  authLimiter: (req, res, next) => next(),
  apiLimiter: (req, res, next) => next(),
  adminLimiter: (req, res, next) => next(),

// Mock external services

jest.mock('../src/services/sms.service');
```

**File: `tests/helpers/test-utils.js`**

## Implementation Steps

### Step 1: Update Test Setup for Speed
Create `tests/setup.js` that:
- Disables rate limiting
- Mocks external services
- Sets up test database helpers

### Step 2: Create Mock Factories
Build reusable mocks for:
- Supabase client responses
- JWT tokens
- Redis client

### Step 3: Write Endpoint Tests (per resource)
For each API resource, create tests covering:
1. **Happy path** - successful requests
2. **Validation errors** - bad input, missing fields
3. **Auth errors** - missing/invalid tokens
4. **Authorization errors** - wrong permissions
5. **Edge cases** - not found, conflicts

### Step 4: Run Tests Rapidly
```bash
# Fast feedback during development
npm run test:integration -- --testPathPattern="user" --silent

# Watch mode
npm run test:watch

# Coverage report
npm run test:coverage
```

## Quick Testing Without Frontend

### Option A: VS Code REST Client
Create `.http` files in `tests/api/`:
```
GET http://localhost:5000/api/v1/products
Authorization: Bearer {{token}}
```

### Option B: Node Script Runner
```javascript
// tests/run-api-test.js
const request = require('supertest');
const app = require('../src/app');

async function testEndpoint() {
  const res = await request(app).get('/api/v1/health');
  console.log('Status:', res.statusCode);
}
testEndpoint();
```

### Option C: Enhanced Jest with Hot Reload
```bash
npx jest --watch --testNamePattern="should register"
```
