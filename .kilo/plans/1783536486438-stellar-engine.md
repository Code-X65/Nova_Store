# Nova Store — Admin Frontend Implementation Contract (COMPLETE)

> **Purpose:** a consolidated, backend-accurate spec of everything the admin frontend must handle. Derived verbatim from the backend route files, validators, controllers, and models (Base mount `/api/v1`; admin routes under `/api/v1/admin`).
> **Global rules the frontend must obey:**
> - Every `/api/v1/admin/*` route is wrapped by `requireAdmin` + `storeContext` + `scopeToStore` (app.js). Admin writes are store-scoped server-side.
> - **CSRF:** `csrfProtection` requires header `x-csrf-token` on every state-changing request (POST/PUT/PATCH/DELETE) made with the **session cookie**. Exempt: Bearer-JWT and safe methods. Frontend must call `GET /api/v1/auth/csrf-token` after login/verify and attach the token to all mutations. (Login/logout/CSRf are exempt.)
> - **Response envelope:** `{ success: true, data, message? }` (or `{ success:false, message }` on error).
> - ⚠️ **Permission vocabulary caveat:** the frontend currently gates some routes on slugs the backend does NOT enforce (`category:manage`, `brand:manage`, `coupon:read`, `staff:read`, `notifications:write`, `inventory:alert`, `order:write`). Backend enforces mostly **role guards** (`requireManager`, `requireOrderStaff`, `requireInventoryStaff`, `requireStoreOwner`) plus slug guards (`product:create|write|delete`, `category:create|write`, `brand:create|write|delete`, `coupon:write`, `shipping:write`, `review:write|read`, `settings:read|write`, `audit:read`, `analytics:read`, `role:manage`, `user:read|write`, `order:read`, `sales:read`, `inventory:read`). Align frontend guards to backend before these pages work.

### Cross-cutting checklist for the frontend
1. **CSRF** injected on every cookie-auth mutation; Bearer used where applicable.
2. **Permission guards aligned** to backend slugs/roles (see caveat at top).
3. **Pagination** everywhere (`page`/`limit` → `total`/`totalPages`).
4. **Soft-delete/archive** semantics (products→archived, categories/brands→`is_active=false`); "show archived" toggles.
5. **Upload flow**: multipart → `/admin/upload` → use returned URL in all image fields.
6. **Date filters**: orders use `dateFrom`/`dateTo`/`userId`; analytics use `from`/`to`. Use exact keys.
7. **Role visibility**: Returns button (manager+), Staff delete (owner), Invitations create (manager, role-restricted).
