# Backend Business Logic Gap Analysis

## Project Overview

Nova Store is a full-featured e-commerce backend using Express.js + Supabase (PostgreSQL) with:

- Product catalog with categories/brands/variants/attributes
- Shopping cart & wishlist
- Checkout with Paystack payments
- Order management with delivery tracking
- Full return/refund lifecycle
- User authentication (email/password + OAuth - Google, Facebook, Apple)
- Admin RBAC system
- Coupons & discounts
- Reviews & ratings
- Shipping zones/rates
- Notifications (email, SMS, in-app)
- Telemetry/analytics

## ⚠️ Partial Gaps / Weaknesses

### 1. Variant Inventory Handling (HIGH)

**File:** `Backend/src/services/cart.service.js`
**Issue:** Cart validates stock against `product.stock_quantity` but variants have their own `stock_quantity` in `product_variants` table which is not used.
**Impact:** Variant-specific stock levels are ignored during cart operations.

### 2. Missing Refund Gateway Integration (MEDIUM)

**File:** `Backend/src/services/order.service.js:492-508`
**Issue:** `process_refund` action sets `return_status`/`refund_status` but doesn't call `PaymentService.refundPayment()` to actually process the refund.
**Impact:** Admin must manually trigger refunds; completion doesn't validate gateway success.

### 3. Incomplete Delivery State Machine (MEDIUM)

**File:** `Backend/src/services/order.service.js:278-303`
**Issue:** `markOutForDelivery` allows transition from current `out_for_delivery` status without validation, and has weak `delivery_status` checking.
**Impact:** Potential for invalid state transitions in delivery flow.

### 4. Missing Idempotency on Order Creation (MEDIUM)

**File:** `Backend/src/services/checkout.service.js:126`
**Issue:** `OrderModel.create()` uses RPC but no idempotency check for duplicate `checkoutSessionId`.
**Impact:** Potential duplicate orders if client retries checkout.

### 5. Missing Guest Cart Merge Stock Validation (MEDIUM)

**File:** `Backend/src/services/cart.service.js:111-141`
**Issue:** `mergeCarts` doesn't validate stock availability when merging.
**Impact:** Items from guest cart may exceed available stock after merge.

### 6. Payment Gateway Abstraction Missing (LOW-MEDIUM)

**File:** `Backend/src/services/payment.service.js`
**Issue:** Direct Paystack coupling; no interface for adding other gateways.
**Impact:** Future payment providers require significant refactoring.

### 7. Attribute Type Validation Edge Cases (LOW)

**File:** `Backend/src/services/attribute.service.js:78-82`
**Issue:** `number`/`boolean` conversions happen without re-validation; Null values cause `String(null)` → "null".

### 8. Review Auto-Detection Missing (LOW)

**File:** `Backend/src/services/review.service.js:70-76`
**Issue:** If `orderId` omitted, `isVerifiedPurchase` defaults to `false` without checking purchase history.

## 📊 Technical Debt

### 1. Supabase RPC Dependency

- Heavy reliance on database functions for atomicity
- Business logic split between JS and SQL layers
- Testing difficulty without DB access

### 2. Service Layer Coupling

- Services directly import each other
- No repository pattern abstraction
- Potential circular dependency risks

### 3. Inconsistent Error Handling

- Different services throw errors inconsistently
- No unified error codes for frontend

## 🚀 Recommendations

### High Priority

1. **Variant Inventory Validation** - Check variant stock in cart operations
2. **Refund Gateway Integration** - Connect `process_refund` to `PaymentService.refundPayment()`
3. **Idempotency for Order Creation** - Check existing order by `checkoutSessionId`

### Medium Priority

4. **Delivery State Machine Guards** - Fix transition validation in delivery methods
5. **Guest Cart Merge Validation** - Add stock checks on merge
6. **Redis Resilience** - Add connection health checks to worker

### Low Priority

7. **Payment Provider Interface** - Abstract gateway operations
8. **Email Template CRUD** - Add admin management endpoints
9. **Loyalty Points Accumulation** - Extend beyond referrals
10. **Bulk Export Endpoints** - Add order/product export

---

## Summary

The backend has substantial functionality with a well-structured service layer. Key gaps: variant inventory tracking ignores variant-specific stock, and `process_refund` doesn't integrate with the payment gateway. Security is well-handled with JWT, RBAC, and middleware protections. Main architectural concern is tight coupling between services and database-layer business logic.
