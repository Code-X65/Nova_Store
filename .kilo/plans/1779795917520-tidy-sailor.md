# Admin Authentication - Single Admin Only Plan (Updated) - IMPLEMENTED

## Overview
Changed the admin authentication to support **only one predefined admin** with no registration capability. The admin has fixed credentials (email: admin@novastore.com, password: Admin.) and exists in the database with all required profile information.

## Changes Made

### 1. Disabled Admin Registration Completely
- **File:** `Backend/src/routes/auth.routes.js` (lines 498-508)
- **Change:** Replaced the admin registration route with a handler that returns 403 Forbidden
- **Details:** 
  - Endpoint `/auth/admin/register` now returns 403 with message "Admin registration is disabled. Only one pre-defined admin account is allowed."
  - All registration attempts are logged for audit purposes via `AuditService.log`

### 2. Enforce Single Admin with Specific Credentials
- **File:** `Backend/src/services/auth.service.js` (lines 218-283)
- **Change:** Modified `adminLogin()` method to validate email is exactly "admin@novastore.com"
- **Details:**
  - First checks if email equals exactly "admin@novastore.com" (case-sensitive)
  - If not, rejects login attempt immediately with "Invalid email or password"
  - Keeps existing validation logic (active status, lockout, verification checks)
  - No password change requirement on first login (as specified)

### 3. Admin Account Requirements in Database
- **File:** `Backend/src/scripts/seed-admin.js` (new file)
- **Purpose:** Seeds the predefined admin user with exact specifications:
  - email: admin@novastore.com (must match exactly)
  - password_hash: bcrypt hash of "Admin."
  - role: 'SUPER_ADMIN' (single super admin as specified)
  - first_name: super
  - last_name: admin
  - is_active: true
  - is_email_verified: true (since no verification flow will be used)
  - is_phone_verified: true (since no verification flow will be used)
  - department: super admin
  - access_level: super_admin
  - phone_number: [optional]
  - home_address: [optional JSON object]

### 4. Admin Controller Changes
- **File:** `Backend/src/controllers/auth.controller.js` (lines 312-320)
- **Change:** Modified `adminRegister` method to return 403 Forbidden for all requests
- **Details:** 
  - Logs attempts for audit purposes
  - Returns consistent 403 response with appropriate message

## Verification
All changes have been implemented and tested for syntax correctness. The seeding script follows the same pattern used in the existing auth service for setting admin roles (create user without role, then update to set role).

## To Deploy
1. Run the seeding script: `node Backend/src/scripts/seed-admin.js`
2. Ensure environment variables are set (SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY)
3. The admin account will then be available with credentials:
   - Email: admin@novastore.com
   - Password: Admin.
