# Plan: Fix YAML Syntax Errors and Runtime Exception in Admin Route Files

## 1. Fix Invalid Tag Formatting (7 files)

In the following files, the file-level `@swagger` block uses an invalid flow sequence `tags: [Name]` followed by nested block mappings `name:` and `description:`. This mixes compact flow style with block style nested mappings, causing "Nested mappings are not allowed in compact mappings" and "Implicit map keys" YAML errors.

Replace the invalid block:
```yaml
/**
 * @swagger
 * tags: [Name]
 *   name: Name
 *   description: ...
 */
```

With valid block syntax:
```yaml
/**
 * @swagger
 * tags:
 *   - name: Name
 *     description: ...
 */
```

## 2. Fix Complex String Parsing in setting.routes.js

In `Backend/src/routes/admin/setting.routes.js`, line 66, the `description` field contains curly braces `{ store_timezone: "Africa/Lagos" }` inside a YAML scalar. While typically valid in a plain scalar, strict YAML parsers can misinterpret the `{` as starting an inline flow mapping. Wrap the description string in double quotes to ensure it is parsed as a literal string.

Replace:
```yaml
description: Map of sub-keys to new values (e.g. { store_timezone: "Africa/Lagos" })
```

With:
```yaml
description: "Map of sub-keys to new values (e.g. { store_timezone: \"Africa/Lagos\" })"
```

## 3. Runtime Exception Investigation

### Root Cause Analysis
The error `UNCAUGHT EXCEPTION! argument handler must be a function` is caused by the YAML parsing failures above. Here is the causal chain:

1. `swagger-jsdoc` is invoked at module load time in `Backend/src/config/swagger.js:36` to parse all `@swagger` JSDoc annotations across every route file.
2. The malformed YAML in the 7 route files causes `swaggerJsdoc(options)` to either throw an uncaught exception or return an invalid/corrupted spec object.
3. In `Backend/src/app.js:218`, the corrupted spec is passed to `swaggerUi.setup(swaggerSpec)`, which returns `undefined` instead of a middleware handler.
4. `app.use('/api-docs', swaggerAuth, swaggerUi.serve, undefined)` then throws `TypeError: argument handler must be a function` because Express received `undefined` instead of a function.

### Cross-Reference Verification
All route handlers were cross-referenced with their controller imports:
- All controller methods referenced in the 8 route files exist and are properly exported.
- Controllers use either `module.exports = new ControllerClass()` or `exports.methodName = ...` patterns; both are compatible with `require()`.
- Middleware functions (`hasPermission`, `hasAnyPermission`, `requireAdmin`) are correctly exported and are valid Express middleware functions.

**Conclusion:** Fixing the YAML syntax errors in steps 1 and 2 will resolve the runtime exception by allowing `swagger-jsdoc` to generate a valid spec, which in turn allows `swaggerUi.setup()` to return a proper middleware handler.
