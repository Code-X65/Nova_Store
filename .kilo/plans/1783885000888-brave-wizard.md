# Fix: Audit trail showing "null" / "(none)" for null initial states

## Root Cause Analysis

The audit log entry for stock adjustments displays `"null"` in the **Changes** drawer and `"(none)"` in the **Summary** text because of a logic error in the diff computation pipeline, not a database schema issue.

### Call chain for a stock adjustment

1. **Controller** (`Backend/src/controllers/inventory.controller.js:86`)
   ```js
   AuditService.log(req, 'inventory.stock_adjusted', 'product', productId,
     null,                                          // ← oldValues is explicitly null
     { quantityChange, reasonCode, notes, variantId }, // ← newValues
     { actionType: 'UPDATE', severity: 'warning' });
   ```
   This is an **action event** (manual adjustment), not a before/after state diff of an entity. The caller correctly passes `null` because there is no prior snapshot of the audit-log payload itself.

2. **AuditService** (`Backend/src/services/audit.service.js:59-61`)
   ```js
   const { delta, summary } = (oldValues || newValues)
     ? computeDelta(oldValues, newValues)
     : { delta: null, summary: null };
   ```
   Because `newValues` is truthy, `computeDelta(null, { quantityChange, reasonCode, notes, variantId })` is invoked.

3. **computeDelta** (`Backend/src/utils/audit-diff.js:57-74`)
   ```js
   const oldMap = oldObj || {};   // null becomes {}
   const newMap = newObj || {};
   const keys = new Set([...Object.keys(oldMap), ...Object.keys(newMap)]);
   // keys = ['quantityChange', 'reasonCode', 'notes', 'variantId']
   for (const key of keys) {
     const before = oldMap[key];  // undefined for every key
     const after = newMap[key];   // actual value
     if (!areEqual(undefined, actualValue)) { // always true
       delta.push({ before: undefined ?? null, after: actualValue ?? null });
     }
   }
   ```
   Every field in `newValues` is treated as a "change" because the empty `oldMap` has none of those keys.

4. **formatValue** (`Backend/src/utils/audit-diff.js:29-30`)
   ```js
   if (value === null || value === undefined) return '(none)';
   ```
   The summary becomes:
   > `Quantity Change changed from (none) to 5; Reason Code changed from (none) to damaged; ...`

5. **Frontend rendering** (`Frontend/src/admin/features/audit/AuditLogs.tsx:29-30`)
   ```js
   function renderDeltaValue(value: unknown): string {
     if (value === null || value === undefined) return 'null';
   ```
   The Changes drawer displays:
   > `QuantityChange: null → 5`

### Why it happens

- **Primary cause**: `computeDelta` is called even when `oldValues` is `null`. It coerces `null` to `{}`, making every field in `newValues` look like a newly introduced field.
- **Secondary cause**: `formatValue` renders `null`/`undefined` as the literal string `'(none)'`, polluting the summary.
- **Classification**: Logic error in the audit diff utility and audit service layer — not a serialization bug or DB schema mismatch.

## Fix Plan

### 1. Update `Backend/src/utils/audit-diff.js`
- Change `formatValue` to return `'—'` (em-dash) instead of `'(none)'` for `null`/`undefined` values. This aligns with the frontend convention and removes the literal "(none)" text from summaries.

### 2. Update `Backend/src/services/audit.service.js`
- Only invoke `computeDelta` when **both** `oldValues` and `newValues` are present and truthy.
- Action events (e.g., `inventory.stock_adjusted`, `rider.created`) pass `null` for `oldValues`; these are action logs, not state diffs, and should not produce a spurious delta.

### 3. Update `Backend/src/models/audit-log.model.js`
- Apply the same guard in the model's `log()` method where it auto-computes delta/summary from raw states.

### 4. Add unit tests for `audit-diff.js`
- Verify `formatValue(null)` returns `'—'`
- Verify `computeDelta(null, { a: 1 })` returns empty delta/summary
- Verify `computeDelta({ a: null }, { a: 1 })` still produces a meaningful transition

### 5. Verify existing inventory tests pass
- Run `npm test` for inventory service to ensure no regressions.

## Expected Outcome

- **Before**: `"Quantity Change changed from (none) to 5; Reason Code changed from (none) to damaged; Notes changed from (none) to ..."` with `null → 5` in Changes drawer.
- **After**: For action events with no prior state, the summary/delta will be `null`/empty (frontend shows `"—"`), eliminating meaningless transitions. For genuine state updates where a field was `null` and became a value, the summary reads `"— to value"` instead of `"(none) to value"`.
