# Catalog Management Auditing Engine — Complete Architecture & Implementation Plan

> **Scope:** Expand the existing `audit_logs` infrastructure into a full-scale Catalog Management auditing engine covering Products, Categories, Attributes, and Brands.
> **Approach:** Three phased deliverables with strict alignment to existing patterns (Express 5 + React 18 + Supabase + Redis + Tailwind, all lowercase snake_case DB, CommonJS backend).

### Gaps to Close

| Gap | Explanation |
|-----|-------------|
| **No catalog-specific audit endpoint** | `/admin/audit` is generic; no `/admin/audit/catalog` with pre-filtered catalog scope, structured change columns, or SKU/friendly-label emphasis |
| **No per-entity change history API** | No endpoint like `/admin/audit/entity/:type/:id` to retrieve the full timeline of changes for a single product/category/brand |
| **No "Clean View" translation layer** | `computeDelta` produces raw `{field, label, before, after}` arrays. There is no service that converts these into role-appropriate descriptive sentences ("Manager X updated the price of Product Y from $10 to $12") |
| **No RBAC-based data redaction on audit reads** | The `findAll` method returns all rows unfiltered. A Manager requesting audit data can currently see Owner-level admin auth events (admin logins, session revocations, system config changes) |
| **Limited event routing for catalog mutations** | Only `catalog.product.deleted` and `catalog.attribute.bulk_changed` fire notifications. Manager-level catalog changes (price updates, category edits, brand updates) do not emit domain events |
| **Inventory transactions conflated with catalog audit** | `inventory_transactions` tracks stock movements. Catalog metadata changes (price, description, status) live in `audit_logs`. There is no unified view that combines both |
| **No version comparison / rollback UI** | `old_values`/`new_values` are stored but there is no UI to compare versions side-by-side or revert |
| **No `device_fingerprint` field** | `device_info` exists as free-form JSON. No structured `device_fingerprint` column |

### 1.1 Database Schema Update

**Migration file:** `Backend/sql/023_catalog_audit_extensions.sql`

The existing `audit_logs` table (migration 020, enriched by 063) is already comprehensive. We need only a modest extension:

```sql
-- ============================================================
-- 023_catalog_audit_extensions.sql
-- Extends the audit schema for catalog-specific traceability.
-- ============================================================

-- 1. Add structured context fields
ALTER TABLE audit_logs ADD COLUMN IF NOT EXISTS device_fingerprint TEXT;
ALTER TABLE audit_logs ADD COLUMN IF NOT EXISTS actor_ip_address INET;
ALTER TABLE audit_logs ADD COLUMN IF NOT EXISTS change_category TEXT CHECK (change_category IN ('metadata','pricing','inventory','media','status','configuration','security'));

-- 2. Add a GIN index for robust JSON delta + JSONB full-text search
CREATE INDEX IF NOT EXISTS idx_audit_logs_delta_gin ON audit_logs USING GIN (delta);
CREATE INDEX IF NOT EXISTS idx_audit_logs_new_values_gin ON audit_logs USING GIN (new_values);
CREATE INDEX IF NOT EXISTS idx_audit_logs_old_values_gin ON audit_logs USING GIN (old_values);

-- 3. Partial index for catalog-only super-fast queries
CREATE INDEX IF NOT EXISTS idx_audit_logs_catalog
  ON audit_logs (created_at DESC)
  WHERE resource_type IN ('product','category','brand','category_attribute','product_variant');

-- 4. Support tsvector for full-text search on summary + action
ALTER TABLE audit_logs ADD COLUMN IF NOT EXISTS search_vector TSVECTOR;
CREATE INDEX IF NOT EXISTS idx_audit_logs_search_vector ON audit_logs USING GIN (search_vector);

-- Maintain search_vector automatically
CREATE OR REPLACE FUNCTION audit_logs_search_vector_trigger() RETURNS trigger AS $$
BEGIN
  NEW.search_vector :=
    setweight(to_tsvector('english', COALESCE(NEW.summary,'')), 'A') ||
    setweight(to_tsvector('english', COALESCE(NEW.action,'')), 'B');
  RETURN NEW;
END
$$ LANGUAGE plpgsql;

DROP TRIGGER IF EXISTS audit_logs_search_vector_trigger ON audit_logs;
CREATE TRIGGER audit_logs_search_vector_trigger
  BEFORE INSERT OR UPDATE ON audit_logs
  FOR EACH ROW EXECUTE FUNCTION audit_logs_search_vector_trigger();

-- 5. Backfill search_vector for existing rows
UPDATE audit_logs
SET search_vector =
  setweight(to_tsvector('english', COALESCE(summary,'')), 'A') ||
  setweight(to_tsvector('english', COALESCE(action,'')), 'B')
WHERE search_vector IS NULL;
```

### 1.2 "Clean View" Translation Service

**New file:** `Backend/src/services/catalog-audit-translator.service.js`

**Purpose:** Converts raw `AuditDeltaEntry[]` arrays into role-appropriate descriptive sentences. This is called by the API layer after enrichment, or by the frontend on demand for richer UX.

**Logic flow diagram:**

```
┌─────────────────────────────────────────────────────────────────┐
│  translateLog(log)                                              │
│                                                                 │
│  Input:  { delta: [{field:'price', label:'Price', before:10,    │
│                      after:12}],                                │
│            action: 'product.updated', actionType: 'UPDATE',     │
│            actorFullName:'Alice', actorRole:'MANAGER',          │
│            resourceName: 'Wireless Earbuds' }                   │
└────────────────────────┬────────────────────────────────────────┘
                         │
          ┌──────────────┼──────────────┐
          │              │              │
          ▼              ▼              ▼
   Build parts[]  Format delta   Join sentence
   ├─ [0] Role    values with   → "Manager Alice
   │  prefix       currency    updated the record for
   └─ "Manager     awareness    Wireless Earbuds — Price
      Alice"                     from $10.00 to $12.00"
          │
          ▼
   Output: "Manager Alice updated the record for Wireless Earbuds — Price from $10.00 to $12.00"
```

**Implementation:**

```javascript
// Backend/src/services/catalog-audit-translator.service.js
const { humanizeAction, humanizeActionType, ACTION_LABELS } = require('../utils/audit-labels');

const FIELD_TEMPLATES = {
  price:             'the {label}',
  cost_price:        'the {label}',
  sale_price:        'the sale {label}',
  stock_quantity:    'the stock quantity',
  status:            'the status',
  name:              'the name',
  description:       'the description',
  primary_image_url: 'the primary image',
  weight:            'the weight',
  is_featured:       'the featured flag',
};

function formatDeltaValue(value, field) {
  if (value === null || value === undefined) return 'empty';
  if (typeof value === 'boolean') return value ? 'enabled' : 'disabled';
  if (typeof value === 'number' && ['price','cost_price','sale_price','stock_quantity','weight'].includes(field)) {
    return value % 1 === 0 ? String(value) : `$${value.toFixed(2)}`;
  }
  if (Array.isArray(value)) return `${value.length} items`;
  if (typeof value === 'object') return JSON.stringify(value);
  return String(value);
}

function formatCleanViewSentence(entry) {
  const template = FIELD_TEMPLATES[entry.field] || `"${entry.label || entry.field}"`;
  const before = formatDeltaValue(entry.before, entry.field);
  const after  = formatDeltaValue(entry.after, entry.field);
  return `${template} from ${before} to ${after}`;
}

function translateLog(log) {
  const parts = [];
  const actionWord = ACTION_LABELS[log.action] || humanizeAction(log.action) || 'modified';

  if (log.actorFullName) {
    const rolePrefix = log.actorRole
      ? `${humanizeActionType(log.actionType)} by ${log.actorRole} ${log.actorFullName}`
      : log.actorFullName;
    parts.push(rolePrefix);
  }

  parts.push(`${actionWord.toLowerCase()} the record for`);
  if (log.resourceName) parts.push(`"${log.resourceName}"`);
  else if (log.resourceSku) parts.push(`SKU ${log.resourceSku}`);
  else if (log.resourceId) parts.push(`ID ${log.resourceId}`);
  else parts.push(`a ${log.resourceType || 'resource'}`);

  if (Array.isArray(log.delta) && log.delta.length > 0) {
    const deltaParts = log.delta.map(d => formatCleanViewSentence(d));
    if (deltaParts.length === 1) parts.push(` — ${deltaParts[0]}`);
    else parts.push(`:\n  - ${deltaParts.join('\n  - ')}`);
  }

  return parts.join(' ');
}

module.exports = {
  translateLog,
  formatCleanViewSentence,
  formatDeltaValue,
};
```

### 1.3 Granular RBAC Data Redaction Middleware

**New file:** `Backend/src/middlewares/audit-redaction.middleware.js`

**Purpose:** Apply role-based filtering + notification on audit log queries before the controller returns data. Prevents MANAGER from seeing STORE_OWNER confidential events, prevents ORDER_STAFF from seeing catalog changes, etc.

**Redaction table:**

| Actor Role | Visible `action_type` | Visible `resource_type` | Extra filter |
|------------|----------------------|------------------------|--------------|
| `STORE_OWNER` | ALL | ALL | None |
| `MANAGER` | CREATE, UPDATE, DELETE, STATUS_CHANGE, OTHER | ALL except `user` (auth) | **Exclude** `action ILIKE 'admin.%'`, `action ILIKE 'user.admin_login%'`, `action_type = 'LOGIN'`. **Exclude** `severity = 'critical'` unless actor_role == requester role or user_id == requester id |
| `ORDER_STAFF` | UPDATE, STATUS_CHANGE | order, order_item, shipment, refund, return | **Exclude** all non-order resource types. **Exclude** CREATE, DELETE. |
| `INVENTORY_STAFF` | CREATE, UPDATE, DELETE, STATUS_CHANGE, OTHER | product, product_variant, inventory_transaction, stock_adjustment | **Exclude** `action ILIKE 'admin.%'`, `action_type = 'LOGIN'`. No order-only restriction (cross-dept read access per RBAC plan). |

**Implementation:**

```javascript
// Backend/src/middlewares/audit-redaction.middleware.js
const { hasRole } = require('./role.middleware');

function getRedactionScope(req) {
  const actor = req.admin;
  if (!actor) return { deny: true, reason: 'unauthenticated' };

  if (hasRole(actor, 'STORE_OWNER')) {
    return { deny: false }; // No filter
  }

  if (hasRole(actor, 'MANAGER')) {
    return {
      deny: false,
      filter: (row) => {
        if (row.action_type === 'LOGIN') return false;
        if (row.action && (row.action.startsWith('admin.') || row.action.startsWith('user.admin_login'))) return false;
        if (row.severity === 'critical' && row.actor_role !== actor.role && row.user_id !== actor.id) return false;
        return true;
      },
      storeOwnerOnlyActionTypes: ['LOGIN'],
      storeOwnerOnlyActions: /^(admin\.|user\.admin_login)/
    };
  }

  if (hasRole(actor, 'ORDER_STAFF')) {
    return {
      deny: false,
      filter: (row) => {
        const allowedTypes = ['order', 'order_item', 'shipment', 'refund', 'return'];
        if (!allowedTypes.includes(row.resource_type)) return false;
        if (!['UPDATE', 'STATUS_CHANGE'].includes(row.action_type)) return false;
        return true;
      }
    };
  }

  if (hasRole(actor, 'INVENTORY_STAFF')) {
    return {
      deny: false,
      filter: (row) => {
        const allowedTypes = ['product', 'product_variant', 'inventory_transaction', 'stock_adjustment'];
        if (!allowedTypes.includes(row.resource_type)) return false;
        if (row.action_type === 'LOGIN') return false;
        if (row.action && row.action.startsWith('admin.')) return false;
        return true;
      }
    };
  }

  return { deny: true, reason: 'no_audit_scope' };
}

function auditRedaction() {
  return (req, res, next) => {
    const scope = getRedactionScope(req);
    if (scope.deny) {
      return res.status(403).json({ success: false, error: 'Access to audit logs is not permitted for your role.' });
    }
    req._auditScope = scope;
    next();
  };
}

module.exports = { auditRedaction, getRedactionScope };
```

```tsx
import { useMyPermissions } from '@/admin/hooks/useMyPermissions';
import { hasPermission, hasAnyPermission } from '@/admin/lib/permissions';
import type { ReactNode } from 'react';

interface Props {
  permission?: string;
  anyOf?: string[];
  children: ReactNode;
  fallback?: ReactNode;
}

export function PermissionGuard({ permission, anyOf, children, fallback = null }: Props) {
  const perms = useMyPermissions();
  if (permission && hasPermission(perms, permission)) return <>{children}</>;
  if (anyOf && hasAnyPermission(perms, ...anyOf)) return <>{children}</>;
  return <>{fallback}</>;
}
```

### 3.2 Catalog-Specific Audit Controller

**New file:** `Backend/src/controllers/admin/catalog-audit.controller.js`

Follow the pattern of `AdminAuditController`. Example for `getCatalogLogs`:

```javascript
const AuditLogModel = require('../../models/audit-log.model');
const { translateLog } = require('../../services/catalog-audit-translator.service');

class CatalogAuditController {
  async getCatalogLogs(req, res, next) {
    try {
      const { page = 1, limit = 20, entityType, entityId, actionType, changeCategory, fromDate, toDate, q } = req.query;
      const result = await AuditLogModel.findAll({
        resourceType: entityType,
        resourceId: entityId,
        actionType,
        fromDate, toDate, q, page, limit,
      });

      // Apply role-based row filter
      if (req._auditScope && req._auditScope.filter) {
        result.logs = result.logs.filter(req._auditScope.filter);
      }

      // Enrich with clean-view translations
      result.logs = result.logs.map(log => ({
        ...log,
        human_readable: translateLog(log),
      }));

      res.status(200).json({ success: true, data: result });
    } catch (error) {
      next(error);
    }
  }

  async getEntityTimeline(req, res, next) {
    try {
      const { type, id } = req.params;
      const { limit = 50, offset = 0 } = req.query;
      const result = await AuditLogModel.findAll({
        resourceType: type,
        resourceId: id,
        page: Math.floor(offset / limit) + 1,
        limit: parseInt(limit),
      });

      if (req._auditScope && req._auditScope.filter) {
        result.logs = result.logs.filter(req._auditScope.filter);
      }

      result.logs = result.logs.map(log => ({
        ...log,
        human_readable: translateLog(log),
      }));

      res.status(200).json({ success: true, data: { entityType: type, entityId: id, history: result.logs } });
    } catch (error) {
      next(error);
    }
  }

  async getCatalogStats(req, res, next) {
    try {
      const { fromDate, toDate } = req.query;
      // Build role-aware stats: STORE_OWNER sees all; MANAGER sees only non-admin actions
      // Implementation follows AuditLogModel.getStats() pattern with additional entity breakdowns
      const stats = await AuditLogModel.getCatalogStats(fromDate, toDate, req._auditScope);
      res.status(200).json({ success: true, data: stats });
    } catch (error) {
      next(error);
    }
  }

  async exportCatalog(req, res, next) {
    // Similar to AuditLogModel.findAllExport() + redaction filter + translateLog mapping
  }
}

module.exports = new CatalogAuditController();
```

## Decisions & Tradeoffs

| Decision | Chosen | Rejected | Rationale |
|----------|--------|----------|-----------|
| **Real-time channel** | SSE (keep existing) | WebSockets/gRPC | App is unidirectional server→admin. SSE natively auto-reconnects, proxied, and already wired. |
| **Redaction timing** | Read-time (row filter after query) | Write-time (separate tables) | Simpler, backward-compatible, no schema duplication. Row-level filter in controller is fast enough with indexes. |
| **Clean View rendering** | Server-side in API response | Client-side only | Saves frontend from re-implementing translation logic and allows non-React consumers (exports). |
| **New table vs. columns** | Columns on `audit_logs` | New `catalog_audit` table | Single source of truth; existing enrichment `enrich()` already handles `resource_type` lookups. |
| **Permission gating style** | Higher-order component (PermissionGuard) | Hidden CSS classes | HOC guarantees elements are absent from DOM, not just visually hidden. |

## Risk Register

| Risk | Mitigation |
|------|-----------|
| `audit_logs` table grows unbounded | Partial indexes keep catalog queries fast; add a monthly `DELETE FROM audit_logs WHERE created_at < now() - interval '2 years'` job if retention policy requires it |
| `translateLog()` performance at scale | Translation is O(n * delta_length). Offset cost by pre-computing `human_readable` in API response (batch) instead of per-row in React render |
| Role check misconfiguration | `requireManager` already typed to allow STORE_OWNER; `auditRedaction` mirrors that exact policy. Unit tests on `getRedactionScope` |
| SSE notification loss | Idempotency via `event_id` unique index + Redis Pub/Sub echo guard prevents duplicates; missed messages are non-critical |
| Frontend PermissionGuard complexity | Start with highest-risk buttons (Delete, Create). Audit remaining unguarded buttons in Phase 2 follow-up |
